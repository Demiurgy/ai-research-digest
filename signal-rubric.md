# Signal Rubric

How the agent decides whether an item is worth sending to Telegram.

## High signal

Send when the post contains at least one strong practical element:

- real workflow
- concrete before/after process change
- agent pipeline
- internal tool or admin UI pattern
- screenshot, demo, architecture, prompt, config, or code
- specific tool stack
- operational lesson from a real team
- measurable result or constraint

Examples:

```text
We replaced weekly product specs with agent-generated prototypes and review queues.
```

```text
Here is our marketing pipeline: news monitoring → relevance scoring → draft generation → human approval → scheduled publishing.
```

```text
This admin workflow uses an agent to prepare actions, but humans approve exceptions before execution.
```

## Medium signal

Send only if the topic is highly relevant.

Typical medium-signal posts:

- useful framing with some concrete details
- product announcement with a visible workflow
- demo without enough explanation
- early pattern that may become important

## Low signal

Do not send.

Low-signal posts include:

- generic AI opinions
- hype without examples
- motivational content
- vague predictions
- announcements without workflow details
- reposts with no added insight
- memes unless they show a useful product/UI pattern

## Main research themes

The digest currently tracks:

- AI-native product work
- PM workflows with agents
- AI marketing and content operations
- internal tools
- admin panels
- back-office workflows
- generative UI
- agent supervision and approval layers
- messaging-first control surfaces
